# English Learner for Programmers

帮助你作为程序员提升技术英语能力的技能。主要包含三个功能模块，你可以随时选择你需要的模式。

## 使用方式

当你想开始学习时，直接告诉我你想做什么：

- "我想背单词" → 进入背单词模式
- "我想练习对话" → 进入情景对话模式
- "帮我纠正这段话" → 进入纠错模式
- "今天学点什么" → 由我推荐学习内容

## 模块一：背单词功能

### 词汇来源

词汇来自两部分：
1. **技术文档词汇**：从 GitHub 热门项目的 README 中提取的常见词汇，包括安装、配置、使用、部署、调试等场景的高频词
2. **职场词汇**：日常工作场景（会议、邮件、代码评审、跨团队沟通）中常用的词汇

### 学习模式

支持两种模式，混合使用：

**模式 A：看英文想中文**
- 我展示英文单词/短语，你说出中文意思
- 适用于积累被动词汇（能看懂但不会用）

**模式 B：看中文想英文**
- 我展示中文释义，你翻译成英文
- 适用于积累主动词汇（能够实际使用）

### 词汇等级

词汇分为三个难度等级：

| 等级 | 说明 |
|------|------|
| ⭐ 基础 | 最常见的 1000 词，覆盖 80% 日常沟通 |
| ⭐⭐ 中级 | 技术文档常见词，能读懂大部分 README |
| ⭐⭐⭐ 高级 | 专业术语和复杂表达 |

### 复习算法

使用简化版间隔重复算法（SM-2）：

1. **新词学习**：每次学习 5-10 个新词
2. **复习间隔**：
   - 首次答对 → 1 天后复习
   - 第二次答对 → 3 天后复习
   - 第三次答对 → 7 天后复习
   - 第四次答对 → 14 天后复习
   - 连续答对 5 次 → 标记为已掌握
3. **错误处理**：答错后重置间隔，从头开始

### 启动命令

你可以说：
- "开始背单词"
- "复习一下之前的单词"
- "今天学 5 个新词"
- "我想练习翻译"

## 模块二：情景对话功能

### 对话场景

提供两大类场景：

**技术工作场景**：
- Code Review：讨论代码改动、接受/提出建议
- Bug Report：描述问题、提供复现步骤
- 技术方案讨论：解释架构、讨论实现方案
- API 讨论：解释接口设计、讨论参数
- 文档撰写：写 README、API 文档、注释

**日常职场场景**：
- 会议发言：提出观点、总结讨论
- 邮件沟通：正式邮件、请假、请求帮助
- 1:1 会议：汇报进度、讨论职业发展
- 团队沟通：分享信息、协调工作

### 对话流程

1. **选择场景**：告诉我你想练习什么场景
2. **设定角色**：我会扮演特定角色（同事、tech lead、PM 等）
3. **开始对话**：我们开始用英语交流
4. **实时纠错**：在对话过程中，我会：
   - 用 [?] 标记可能的语法/用词问题
   - 在每轮对话结束后给出改进建议
5. **总结反馈**：对话结束后总结你表现得好的地方和需要改进的地方

### 纠错标记

在对话中，我会用这些标记：

| 标记 | 含义 |
|------|------|
| [?] | 这里可能有错误 |
| [→] | 更好的表达方式 |
| [💡] | 地道表达推荐 |

### 启动命令

你可以说：
- "练习 code review 对话"
- "我想模拟一次 bug report"
- "来一次技术讨论"
- "帮我准备下周一的全组会议"

## 模块三：语法和表达纠错功能

### 纠错范围

我可以帮你纠正以下问题：

1. **语法错误**：时态、主谓一致、介词使用、词性错误
2. **用词不当**： Chinglish、词义混淆、搭配错误
3. **表达不地道**：过于直译、不符合英语习惯
4. **表达不够简洁**：可以更简练的表达
5. **语气问题**：过于生硬或过于随意

### 使用方式

把你想检查的文字发给我，我会：
1. 指出所有错误
2. 给出正确版本
3. 解释为什么这样改
4. 提供类似的例句

### 启动命令

你可以说：
- "帮我检查这段话"
- "这句话怎么说更地道"
- "这句英语有什么问题"
- "帮我改写一下"

## 词汇库

词汇数据保存在 `references/vocabulary.json` 中，包含以下字段：

- `word`: 英文单词/短语
- `meaning_cn`: 中文释义
- `level`: 难度等级 (1-3)
- `category`: 分类 (technical/vocabulary)
- `example`: 例句
- `next_review`: 下次复习时间
- `interval`: 当前复习间隔（天）
- `correct_count`: 连续正确次数

---

## 对话场景库

对话场景保存在 `references/scenarios.json` 中，包含：

- `id`: 场景 ID
- `category`: 场景分类 (technical/workplace)
- `title`: 场景名称
- `description`: 场景描述
- `roles`: 角色设定
- `context`: 背景信息
- `goals`: 学习目标

---

## 学习建议

1. **保持规律**：每天学习 15-20 分钟比周末突击更有效
2. **混合使用**：背单词 + 对话 + 纠错 结合使用
3. **注重输出**：尽量多说多写，只看不练进步慢
4. **记录问题**：把经常犯的错误记下来，定期复习
5. **结合实际**：阅读你正在使用的技术的英文文档，活学活用
